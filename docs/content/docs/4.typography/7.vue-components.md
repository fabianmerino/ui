Vue components in Nuxt UI's typography system provide specialized functionality beyond standard markdown elements. These components enable rich, interactive content authoring using the [MDC syntax](https://content.nuxt.com/docs/files/markdown#mdc-syntax), perfect for creating engaging documentation, tutorials, and content-rich applications.

## Usage

Vue components are used within markdown content using the MDC syntax. They support props, slots, and full markdown content within their bodies:

```mdc
::callout{icon="i-lucide-info" color="blue"}
This is a callout with an icon and custom color. You can use **markdown** formatting inside.
::
```

## Component Categories

### Content Enhancement

Components that add visual interest and improve content readability:

- **[Badge](/docs/typography/badge)** - Display version numbers, status labels, or tags
- **[Callout](/docs/typography/callout)** - Eye-catching context boxes with icons and colors  
- **[Card](/docs/typography/card)** - Highlighted content blocks with optional links
- **[CardGroup](/docs/typography/card-group)** - Grid layouts for organizing multiple cards

### Code & Technical Content

Specialized components for presenting code and technical information:

- **[CodeGroup](/docs/typography/code-group)** - Tabbed code blocks for multiple examples
- **[CodePreview](/docs/typography/code-preview)** - Live previews alongside code examples
- **[CodeTree](/docs/typography/code-tree)** - File structure visualization with code
- **[CodeCollapse](/docs/typography/code-collapse)** - Collapsible code sections for long examples

### Interactive Elements

Components that respond to user interaction:

- **[Accordion](/docs/typography/accordion)** - Expandable content sections
- **[Collapsible](/docs/typography/collapsible)** - Simple show/hide content areas
- **[Tabs](/docs/typography/tabs)** - Tabbed content organization

### Documentation Tools

Components specifically designed for documentation and API references:

- **[Field](/docs/typography/field)** - Parameter and property documentation
- **[FieldGroup](/docs/typography/field-group)** - Grouped parameter lists
- **[Steps](/docs/typography/steps)** - Sequential instruction guides
- **[Icon](/docs/typography/icon)** - Inline icon display
- **[Kbd](/docs/typography/kbd)** - Keyboard shortcut visualization

## Best Practices

### Component Composition

Combine components strategically for maximum impact:

```mdc
::card-group
::card{title="Documentation" icon="i-lucide-book-open" to="/docs"}
Comprehensive guides and API references.
::

::card{title="Examples" icon="i-lucide-code" to="/examples"}
Live examples and code samples.
::

::card{title="Community" icon="i-lucide-users" to="/community"}
Join our growing community of developers.
::
::
```

### Content Organization

Use structural components to organize complex information:

```mdc
::field-group
::field{name="title" type="string" required}
The main heading for your page or component.
::

::field{name="description" type="string"}
Optional description providing additional context.
::

::field{name="icon" type="string"}
Icon name from the available icon sets.
::
::
```

## Extending Components

Vue components in Nuxt UI are designed to be extensible. You can:

- **Customize styling** through the theming system
- **Add new variants** via app configuration
- **Create custom components** following the same patterns
- **Override default behavior** through component props

::tip
All Vue components support the full Nuxt UI theming system and automatically adapt to your color mode, typography, and spacing preferences.
::
